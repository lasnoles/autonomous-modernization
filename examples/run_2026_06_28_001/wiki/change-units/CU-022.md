## Rationale

Spring 3.x best practice and testability favor constructor injection over
`@Autowired` field injection (immutable required dependencies, no reflection
needed in tests, clearer dependency graph). This cleanup runs in wave 3 alongside
the seam work so the freshly extracted payments classes adopt the pattern too.
