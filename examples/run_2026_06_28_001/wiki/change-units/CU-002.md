## Rationale

The payments code path pulled the external provider SDK and several HTTP/JSON
libraries straight into `com.acme.billing` core packages, creating a tangle that
would have made the wave-3 seam extraction ([CU-021](CU-021.md)) far riskier.
This CU pulls those dependencies behind a single `payments` module boundary so
that the later seam can be introduced with a clean cut. It is sequenced **first**
(wave 1) precisely so that every downstream change operates on an already-
decoupled module.

## Targets (graph GIDs)

- `billing-svc::Package::com.acme.billing.payments`

## Strategy / recipe

```yaml
strategy:
  preferred: recipe
  recipe: org.openrewrite.maven.RemoveRedundantDependencyVersions + org.openrewrite.java.dependencies.ModuleIsolation
  fallback: manual
```

Recipe groups the provider SDK and transport libraries into the `payments`
module and removes their leakage from `com.acme.billing` core packages.
