## Rationale

The dependency scan flagged `log4j-core 2.14` — vulnerable to
**CVE-2021-44228 (Log4Shell)**, a critical unauthenticated remote-code-execution
via JNDI lookup. This CU upgrades Log4j to **2.17.x**, which removes the
vulnerable lookup behavior. Sequenced in wave 1 because a critical RCE should be
closed before any other change ships.

## Targets (graph GIDs)

- `billing-svc::BuildFile::billing-svc/pom.xml`
- `billing-svc::Dependency::org.apache.logging.log4j:log4j-core`

## Strategy / recipe

```yaml
strategy:
  preferred: recipe
  recipe: org.openrewrite.java.dependencies.UpgradeDependencyVersion (log4j-core → 2.17.x)
  fallback: manual
```
