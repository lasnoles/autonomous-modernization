## Rationale

Spring Boot 3 requires Jakarta EE 9+ namespaces. Every `javax.*` import in the
service must move to `jakarta.*` before [CU-018 (Spring Boot 3.x upgrade)](CU-018.md)
can land. This is a pure namespace migration with no behavioral intent.

## Targets (graph GIDs)

- `billing-svc::Package::com.acme.billing`
- `billing-svc::Class::com.acme.billing.InvoiceService`
- `billing-svc::Class::com.acme.billing.web.InvoiceController`

## Equivalence

- **Level:** **behavioral**
- **Tests:** existing + generate-if-missing
- **Replay:** false

The migration is a deterministic namespace rewrite; behavioral equivalence is
proven by the existing suite passing unchanged plus regenerated coverage for two
controllers that lacked it. No public-API signature changes.

## PRESERVES rules

  validation, tax computation, and charge orchestration are untouched; only
  namespaces of annotations/types moved.
- Public HTTP contract of `InvoiceController` — unchanged.
